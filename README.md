# Ronaq Dashboard (لوحة تحكم رونق)

نظام إدارة المخزون والمبيعات المتكامل لمصنع وملصقات **رونق (Ronaq)**.

## مميزات النظام
- **إدارة المخزون**: متابعة حالة الملصقات والكميات المتوفرة وتنبيهات انخفاض المخزون.
- **إدارة الطلبات**: دورة حياة الطلبات كاملة من الإنشاء والتجهيز والشحن حتى التوصيل.
- **الحسابات والأرباح**: تسجيل المصروفات، الأرباح، الإيرادات وحساب الأرباح الصافية (خاص بالمدراء).
- **أدوار المستخدمين**: صلاحيات مخصصة (المدراء Amr + Khaled وموظفي الإنتاج).
- **تصميم متميز**: واجهة مستخدم كاملة باللغة العربية (RTL) متطابقة مع التصاميم المعتمدة وتدعم الهواتف والأجهزة اللوحية.

## هيكلية المشروع (Monorepo)
- `/client`: تطبيق الواجهة الأمامية مبني باستخدام React + Vite + TailwindCSS.
- `/server`: خادم واجهة برمجة التطبيقات مبني باستخدام Node.js + Express + Supabase SDK.
- `/database`: ملفات تهجير البيانات (SQL Migrations) وقواعد البيانات لـ Supabase/PostgreSQL.

## التشغيل المحلي (Development)

### المتطلبات الأساسية
- Node.js (v18+)
- حساب Supabase مفعل وقاعدة بيانات جاهزة.

### خطوات الإعداد والتشغيل
1. **تهيئة البيئة**:
   قم بإنشاء ملف `.env` في المجلد الرئيسي وفي مجلد `/server` ومجلد `/client` بناءً على ملفات `.env.example` المرفقة.

2. **تثبيت الحزم**:
   قم بتشغيل الأمر التالي في المجلد الرئيسي للمشروع لتثبيت جميع الاعتمادات في الواجهة الأمامية والخلفية دفعة واحدة:
   ```bash
   npm run install:all
   ```

3. **تشغيل المشروع**:
   لتشغيل الخادم والتطبيق معاً في نفس الوقت بوضع التطوير:
   ```bash
   npm run dev
   ```
   سيتم تشغيل:
   - تطبيق React على الرابط: `http://localhost:5173`
   - خادم API على الرابط: `http://localhost:3001`

4. **تشغيل الفحص والتحقق (Linting & Compilation)**:
   ```bash
   npm run typecheck
   ```

## النشر (Deployment)
- يتم نشر خادم API على **Vercel** باستخدام ملف `vercel.json` المضمن.
- يتم نشر تطبيق React على **Vercel** أو **Netlify** كـ Static SPA.
